04 / 12

Local Device Storage

The application stores a small amount of data locally on your device to provide a consistent user experience across sessions. This data remains exclusively on your device and is never transmitted externally.

📱 Android — SharedPreferences

On Android, Igbo Calendar uses Android's SharedPreferences system to store your selected colour theme (Amber or Green). This is a standard key-value store that is sandboxed to the app and inaccessible to other apps on your device. Cloud backup is explicitly disabled (android:allowBackup="false"), meaning this preference is never uploaded to Google Drive or any other service.

🍎 iOS — UserDefaults

On iOS, the same theme preference is stored using UserDefaults. This is an on-device key-value store private to the application sandbox. iCloud sync for this preference is not enabled.

🌐 Web — No Persistent Storage

The web version of Igbo Calendar may use your browser's localStorage or sessionStorage to remember your theme preference during and across browsing sessions. No cookies are used for tracking purposes. No data is transmitted to our servers from the web version.

You can clear this locally stored data at any time by uninstalling the app (Android/iOS) or clearing your browser's site data (web). No data will be lost from any server, because none was ever sent.

06 / 12

Internet Access

The Android and iOS versions of Igbo Calendar do not request or use internet access. The INTERNET permission is not declared in the Android application manifest. The app operates in a fully offline mode at all times.

All core functionality — Igbo/Gregorian date conversion, market day calculation, the 13-month calendar structure, week (Izu) determination, and all educational content about Igbo months, weeks, market days, and festivals — is bundled within the application and computed locally on your device.

The web version at calendar.vodatrox.com operates as a standard static website. Standard web server logs (IP address, request timestamp, browser type) may be generated by the hosting provider as part of normal server operation. These logs are not used for user profiling or tracking, and are not shared with third parties except where required by law or necessary for server security.

Data Security

Because Igbo Calendar does not transmit any user data over the internet, the most significant class of data breach risk — unauthorised network interception — does not apply to this application.

The sole piece of data stored by the application (your theme preference) is protected by your device's operating system sandbox. On both Android and iOS, application data is sandboxed and inaccessible to other applications without explicit system-level compromise (such as a rooted/jailbroken device). We recommend keeping your device's operating system up to date to benefit from the latest security patches.

We take the security of our application code seriously. The Android release build is minified and resource-shrunken using ProGuard to reduce the attack surface. We do not hard-code sensitive credentials into the application because no server-side communication takes place.

Data Retention

The theme preference stored locally on your device is retained for as long as the application remains installed. It is automatically and permanently deleted when you uninstall the application from your device. No copy of this preference is held on our servers at any time.

Standard web server access logs (IP address, user-agent, request timestamp) generated by the web hosting provider are typically retained for a period of up to 90 days for security and operational monitoring purposes, after which they are automatically purged. These logs are not linked to individual users and are not used for any form of profiling.
